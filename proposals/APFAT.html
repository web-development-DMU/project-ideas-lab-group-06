A Penny For a Thought

proposed by: Thanaa Sulum

Project Description

This projects aims to create a web application, that encourages users to take control of their finances, by allowing them to track their expenses, record their daily spending, and reflect on the emotional and psychological aspect of their spending habits. The application will also allow users to set savings goals, and track their progress towards their goals, as well as provide insights and analytics about their spendings habits. Rather than simply being a budgeting dashboard, the application promotes self-awareness and accountability.

This web application will be designed to be user friendly, there wont be any administrators, as users would completely manage their own accounts.

User requirements:

  • User should be able to create their own profile
  • User should be able to import and export data
  • User should be able to add expenses and income
  • User should be able to make reflections on their expenses
  • User should be able to set and track savings goals
  • User should be able to view insights about their spending habits

Ideas for a basic database structure:

  • user: email, user_id (PK), username, password_hash, created_at
  • saving_goals: created_at, completion_status, target_amount, goal_name, user_id (FK), goal_id(PK)
  • Saving_entries: saving_entry_id, saving_id(FK), goal_id(FK), amount, date
  • transaction: reflection_note,mood , created_at, category, type(expense/income), transaction_date, amount, user_id(FK), description
  • daily_summary: daily_total_id(PK), user_id(FK), date, target_spent_total, created_at

Data requirements

  • User authentication data (accounts and privacy).
  • Transaction records (amount, category, date, optional reflection).
  • Savings goals and savings entries to track progress.
  • Insights data to provide feedback on spending habits.
