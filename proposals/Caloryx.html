Caloryx

Proposed By: Joshua Moody, Alfie Tones

Overview

The idea for this application is to create a functional web app that allows users to track their caloric intake and the different amount of macronutrients that they consume, combining the total to display for them. The app will allow users to input food they have eaten and the amount of calories and macronutrients in that food, and then it will calculate the total calories and macronutrients consumed for the day. The app will also allow for users to set workouts they have completed and the calories burned from those workouts, which will then be subtracted from the total calories consumed to give the user a net calorie count for the day. The app will also have a feature that allows users to set goals for their caloric intake and macronutrient consumption, and it will provide feedback on how well they are meeting those goals. This combines both a calorie tracking and a workout tracking app into one, making it easier for users to keep track of their overall health and fitness goals.

It will need a database to store user information, food items, and workout data. It will also need a user interface to allow users to input their data and view their progress. Theres a possibility that it needs an admin system to manage user accounts and content, such as adding new food items or managing user data.

Features

  • Track caloric intake and macronutrient consumption
  • Set and track daily goals for calories and macronutrients
  • Log completed workouts and calories burned
  • Calculate net calorie count for the day
  • Admin panel for managing users and content
    • Add new food items to the database
    • Manage user accounts and data

List of Possible Names

The current name is Caloryx, but other possible names include:

  • NutriTrack
  • MacroMind
  • CalorieCounter
  • FitFuel

Diffent Types of Users

The different types of users for this application include:

  • Regular Users - Individuals who want to track their caloric intake and macronutrient consumption
  • Admin Users - Users who manage the application, including adding new food items and managing user accounts
  • Guest Users - Users who can view public information but cannot make changes
  • Premium Users - Users who have access to additional features and functionality

User Requirements

Users of this application should be able to:

  • Log food items and their nutritional information
  • Set daily goals for calories and macronutrients
  • Track completed workouts and calories burned
  • View their net calorie count for the day
  • Access premium features if they are premium users

Admin users should be able to:

  • Add new food items to the database
  • Manage user accounts and data
  • Access analytics and reports on user activity

Technical Requirements

The technical requirements for this application include:

  • A database to store user information, food items, and workout data

    • The database structure could possible look like this:

    • Users table: id, username, email, password, user_type (regular, admin, guest, premium)
    • Food items table: id, name, calories, protein, carbs, fat
    • Workouts table: id, user_id, workout_name, calories_burned
    • Goals table: id, user_id, daily_calorie_goal, daily_protein_goal, daily_carb_goal, daily_fat_goal
  • A user interface for inputting data and viewing progress
  • Authentication and authorization system for managing user accounts and access levels
  • Integration with third-party APIs for nutritional information and workout tracking (optional)

Potential Challenges

Some potential challenges in developing this application include:

  • Ensuring data accuracy and reliability for food items and workouts
  • Designing a user-friendly interface that is easy to navigate
  • Implementing a secure authentication and authorization system
  • Integrating with third-party APIs for nutritional information and workout tracking (if applicable)
  • Managing user data and ensuring privacy and security

Data Requirements

The data requirements for this application include:

  • User information (username, email, password, user type)
  • Food items and their nutritional information (name, calories, protein, carbs, fat)
  • Workout information (user_id, workout_name, calories_burned)
  • Goal information (user_id, daily_calorie_goal, daily_protein_goal, daily_carb_goal, daily_fat_goal)
