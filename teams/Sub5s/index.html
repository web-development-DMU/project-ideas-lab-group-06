Application Ideas

Me and Josh have decided to go with an application we call Caloryx. [found here.]

Caloryx is an application that allows users to track their calorie intake and expenditure. We plan to keep the application simple, and thus user-friendly. The application will allow users to input their meals and exercises, and will then calculate the total calories consumed and burned. The application will also provide users with a daily calorie goal based on their age, gender, weight, and activity level. We're hoping that this application will help users (including ourselves) to keep on track of thier fitness goals!

Documentation

Database Structure.

We plan to use a simple database structure for our application. We will have a users table that will store user information such as username, password, age, gender. We will also have a meals table that will store information about the meals that users input, such as the name of the meal, the calories, and the user who inputted it. Finally, we will have an exercises table that will store information about the exercises that users input, such as the name of the exercise, the calories burned, and the user who inputted it.

For more information visit... [database structure]

User Interface Design

We plan to keep the user interface of our application simple and user-friendly. We will use a clean and minimalistic design, with a focus on functionality. The main page of the application will display the user's daily calorie goal, as well as their total calories consumed and burned for the day. Users will be able to easily navigate to the meals and exercises pages to input their information.

Routes

We plan to have the following routes for our application:

  • / - Home page
  • /login - Login page
  • /register - Registration page
  • /dashboard - User dashboard
  • /meals - Meals input page
