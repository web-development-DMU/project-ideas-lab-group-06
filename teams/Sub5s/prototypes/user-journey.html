Caloryx – User Journey

Walkthrough of the main screens from a user's and admin's perspective.

User Journey

1 Home page

The user arrives at the landing page and sees the app's main features. They can choose to register for a new account or log in to an existing one.

↓

2 Register

A new user fills in their username, email, password, age, and gender. On success they are taken straight to their dashboard.

↓

3 Log in

Returning users enter their username and password. Invalid credentials show an inline error message.

↓

4 Dashboard

The dashboard shows today's calories consumed, burned, and remaining, along with macronutrient progress bars and a summary of logged meals and exercises.

↓

5 Meals list

All meals logged for the selected day are shown here. The user can search or filter by date, and click any entry to see its full details.

↓

6 Meal detail

Shows the full nutritional breakdown for a single meal. The user can edit or delete the entry from this page.

↓

7 Exercises list

All exercises logged for the selected day, filterable by date or type. Each card shows name, duration, and calories burned.

↓

8 Exercise detail

Full details for a single exercise entry including duration, intensity, and calories burned. Can be edited or deleted.

↓

9 Settings

Users can update their profile information and adjust their daily calorie and macro targets. Changes are saved with a confirmation message.

Admin Journey

10 Admin dashboard

Admins land on an overview showing platform-wide stats: total users, meals and exercises logged today, and food database size.

↓

11 Manage users

A searchable table of all registered users. Each row links to that user's edit page.

↓

12 Edit user

Admins can update a user's profile fields, change their role, or delete the account entirely from the danger zone at the bottom.

↓

13 Food database

Lists all food items with their nutritional values. Admins can search, add new items, or click through to edit an existing one.

↓

14 Edit food item

A form pre-filled with an item's nutritional data. Admins can correct values or delete the item if it is a duplicate.

↓

15 Analytics

Platform-level stats including active users, average meals logged, popular foods, and a registration chart for the past week.

↓

16 Reports

A list of available system reports. Each report shows when it was last generated and any warnings (e.g. missing nutritional data).

↓

17 System settings

Controls global platform behaviour: default calorie goals for new users, feature toggles (maintenance mode, registrations, backups), and backup retention period.
