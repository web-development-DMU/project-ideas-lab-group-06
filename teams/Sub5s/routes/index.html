Caloryx Routes Documentation

Overview

This documentation outlines all the routes for the Caloryx application. Routes are organized by category and request method. GET requests are read-only (safe and idempotent) operations for displaying data. POST requests modify server state.

Select a category below to view detailed route information:

Route Categories

Public Routes

User-accessible routes for authentication, dashboard, and managing meals and exercises. These routes require user login.

Prototypes

Static HTML prototype pages have been created for every GET route listed in this documentation. Each prototype shows a realistic example of what the page will look like, using hardcoded sample data.

A narrative user journey document describes the full flow through the interface — from a new visitor arriving at the home page, through registration, daily use, and the admin panel. Each step embeds the relevant prototype page so you can see the screens in context.
