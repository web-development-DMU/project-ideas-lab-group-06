The Database Structure

Tables needed for "A Penny For A Thought"

Ideas for a basic database structure:

  • User: email, user_id (PK), username, hashedPassword, created_at
  • Saving_goals: created_at, completion_status, target_amount, goal_name, user_id (FK), goal_id(PK)
  • Saving_entries: saving_entry_id, goal_id(FK), amount, date
  • Transactions: transaction_id(PK),mood, reflection_note, created_at, category, type(expense/income), transaction_date, amount, mood, user_id(FK), description
  • Daily_summary: daily_total_id(PK), user_id, date, target_spent_total, created_at

PK = Primary Key and FK = Foreign Key

Mood and reflections_note are fields that support insight queries.

Fields and relationships:

There will be just one to many relationships between the tables.

  • A user has many saving entries
  • A user has many saving goals
  • A savings goal has many savings entries
  • A user has many transactions
  • A user has many daily summaries

Example SQL Schema Design

  • INSERT INTO transactions ...
  • SELECT * FROM transactions WHERE user_id = ? ORDER BY transaction_date DESC;
  • UPDATE transactions SET ... WHERE transaction_id = ? AND user_id = ?;
  • DELETE FROM transactions WHERE transaction_id = ? AND user_id = ?;
  • Monthly totals: SELECT SUM(amount) ... WHERE type='expense' AND date LIKE 'YYYY-MM%';
  • Goal progress: SELECT SUM(amount) FROM savings_entries WHERE goal_id = ?;
