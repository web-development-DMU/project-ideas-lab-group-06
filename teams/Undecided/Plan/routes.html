HTTP Interactions

Home Page

  • Route: GET /, the server displays the home page for the application, which introduces the app and provides navigation links to register, log in, or access the dashboard if the user is already authenticated.
  • Data required: None.
  • Database Side effects: None.
  • SQL: No SQL required.
  • Response: Returns an HTML page.

Registration Page

  • Route: GET /register, the server displays the registration form for new users.
  • Data required: None.
  • Database Side effects: None.
  • SQL: No SQL required.
  • Response: Returns an HTML page with the registration form.

Register New User

  • Route: POST /register, creates a new user account.
  • Data required: username, email, and password.
  • Database Side effects: Creates a new record in the users table.
  • SQL: INSERT INTO users (username, email, hashedPassword) VALUES (?, ?, ?);
  • Response: Redirects to /login on success, or returns the registration page HTML with validation errors.

Login Page

  • Route: GET /login, displays the login form.
  • Data required: None.
  • Database Side effects: None.
  • SQL: No SQL required.
  • Response: Returns the login page HTML.

Login User

  • Route: POST /login, authenticates the user and creates a session.
  • Data required: username and password.
  • Database Side effects: Creates a new record in the sessions table.
  • SQL: SELECT * FROM users WHERE username = ?; INSERT INTO sessions (id, user_id) VALUES (?, ?);
  • Response: Redirects to /dashboard on success, or returns the login page HTML with validation errors on failure.

Logout User

  • Route: POST /logout, logs the user out by removing their session.
  • Data required: session cookie, session id.
  • Database Side effects: User's session is removed from the sessions table.
  • SQL: DELETE FROM sessions WHERE id = ?;
  • Response: Redirects to / or /login.

Dashboard

  • Route: GET /dashboard, displays the user dashboard.
  • Data required: session cookie, user id.
  • Database Side effects: None.
  • SQL: SELECT * FROM transactions WHERE user_id = ?; SELECT * FROM saving_goals WHERE user_id = ?;
  • Response: Returns dashboard HTML.

View Transactions

  • Route: GET /transactions, displays all user transactions.
  • Data required: session cookie, user id.
  • Database Side effects: None.
  • SQL: SELECT * FROM transactions WHERE user_id = ? ORDER BY date DESC;
  • Response: Returns the transactions page HTML.

New Transaction Form

  • Route: GET /transactions/new, displays the add transaction form.
  • Data required: session cookie, user id.
  • Database Side effects: None.
  • SQL: No SQL required.
  • Response: Returns the form HTML.

Create Transaction

  • Route: POST /transactions, adds a new transaction.
  • Data required: amount, category, date, mood, reflection_note.
  • Database Side effects: Creates a new record in the transactions table.
  • SQL: INSERT INTO transactions (user_id, amount, category, date, mood, reflection_note) VALUES (?, ?, ?, ?, ?, ?);
  • Response: Redirects to /transactions.

View Single Transaction

  • Route: GET /transactions/:transactionId, displays one transaction.
  • Data required: transactionId, user id.
  • Database Side effects: None.
  • SQL: SELECT * FROM transactions WHERE transaction_id = ? AND user_id = ?;
  • Response: Returns transaction detail HTML.

Delete Transaction

  • Route: POST /transactions/:transactionId/delete, deletes a transaction.
  • Data required: transactionId, user id.
  • Database Side effects: Removes a record from the transactions table.
  • SQL: DELETE FROM transactions WHERE transaction_id = ? AND user_id = ?;
  • Response: Redirects to /transactions.

View Saving Goals

  • Route: GET /saving-goals, displays all saving goals.
  • Data required: session cookie, user id.
  • Database Side effects: None.
  • SQL: SELECT * FROM saving_goals WHERE user_id = ?;
  • Response: Returns saving goals HTML.

Create Saving Goal

  • Route: POST /saving-goals, creates a new saving goal.
  • Data required: goal_name, target_amount.
  • Database Side effects: Creates a new record in the saving_goals table.
  • SQL: INSERT INTO saving_goals (user_id, goal_name, target_amount) VALUES (?, ?, ?);
  • Response: Redirects to /saving-goals.

View Single Saving Goal

  • Route: GET /saving-goals/:goalId, displays one saving goal.
  • Data required: goalId, user id.
  • Database Side effects: None.
  • SQL: SELECT * FROM saving_goals WHERE goal_id = ? AND user_id = ?; SELECT * FROM saving_entries WHERE goal_id = ?;
  • Response: Returns saving goal detail HTML.

Add Saving Entry

  • Route: POST /saving-goals/:goalId/entries, adds money to a saving goal.
  • Data required: goalId, amount, date, description.
  • Database Side effects: Creates a new record in the saving_entries table.
  • SQL: INSERT INTO saving_entries (goal_id, amount, date, description) VALUES (?, ?, ?, ?);
  • Response: Redirects to /saving-goals/:goalId.

Delete Saving Entry

  • Route: POST /saving-entries/:entryId/delete, deletes a saving entry.
  • Data required: entryId.
  • Database Side effects: Removes a record from the saving_entries table.
  • SQL: DELETE FROM saving_entries WHERE savings_entry_id = ?;
  • Response: Redirects to the saving goal page.

Insights Page

  • Route: GET /insights, displays financial insights.
  • Data required: user id.
  • Database Side effects: None.
  • SQL: SELECT category, SUM(amount) AS total_spent FROM transactions WHERE user_id = ? GROUP BY category;
  • Response: Returns the insights HTML page.
