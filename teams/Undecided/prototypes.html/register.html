User journey through "A Penny For A Thought"

The user lands on the home page, which has options to either log in or register. If the user chooses to register, they would be directed to a page through a GET request to the registration page, where they will fill in the form requiring their username, email and password.

This is route for the registration Page:

  • Route: GET /register, the server displays the registration form for new users.
  • Data required: None.
  • Database Side effects: None.
  • SQL: No SQL required.
  • Response: Returns an HTML page with the registration form.

If the user, however, already has an account then they would be sent to the login page, where they only need to input their username and password, to access the dashboard where they can create transaction, savings entry as well as accessing their insights and goals etc.

This is the route for the login Page:

  • Route: GET /login, displays the login form.
  • Data required: None.
  • Database Side effects: None.
  • SQL: No SQL required.
  • Response: Returns the login page HTML.

note: There are many more routes, however only two examples are currently shown to demonstrate how the app may work through HTTP requests.
