Ideas — List 1

  • Lost and found website
    • User requirements
      • Report lost item with photo, location, and contact.
      • Search and filter items by category, date, and location.
      • Claim verification process for owners (evidence upload).
    • Fields used for queries
      • items: id, title, description, photo_url, status (lost/found/claimed), reporter_id, claimer_id, created_at, updated_at, claimed_at
      • locations: id, item_id, location_text
      • users: id, name, email, phone, verified
      • claims: id, item_id, user_id, evidence_url, status, created_at
    • User specifications
      • Roles: guest, user, admin.

Ideas — List 2

  • Recipe sharing website
    • User requirements
      • Create, edit, and publish recipes with ingredients and steps.
      • Rate/comment on recipes and save favorites.
      • Search by ingredient, cuisine, dietary tags.
    • Fields used for queries
      • recipes: id, title, description, author_id, servings, prep_time, cook_time, difficulty, photo_url, created_at, updated_at
      • ingredients: id, name, normalized_name
      • recipe_ingredients: id, recipe_id, ingredient_id, quantity, unit, notes
      • ratings: id, recipe_id, user_id, score, comment, created_at
      • favorites: id, user_id, recipe_id, created_at
      • tags: id, name; recipe_tags: recipe_id, tag_id
    • User specifications
      • Roles: guest, user, moderator, admin.

Ideas — List 3

  • Social meetup finder
    • User requirements
      • Create and browse events with date, time, location, and capacity.
      • RSVP and messaging for attendees; event recommendations.
      • Map view and filters by interest, date, distance.
    • Fields used for queries
      • events: id, title, description, host_id, start_time, end_time, capacity, location_text, address, is_public, status, created_at, updated_at
      • rsvps: id, event_id, user_id, status (attending/interested/cancelled), created_at
      • users: id, name, email, profile_photo, interests, verified
      • event_tags: event_id, tag_id; tags: id, name
      • messages: id, event_id, sender_id, content, created_at
    • User specifications
      • Roles: guest, user, moderator, admin.
